Creating and Editing Assistants

This guide walks you through every field in the New Assistant form. By the end you'll know how to pick the right model, tweak its behaviour, and enrich it with documents so it can answer smarter questions.

1. Name

The Name field is purely cosmetic. It's how the assistant will appear in the list of assistants and in the chat UI. Keep it short, memorable and descriptive (e.g., Java Bot, Diagram Maker).

2. Shared

When checked, the assistant can be used by all users. If you only want to use it yourself, leave it unchecked. If you intend to use this assistant in a workflow that you want to share, you must share the assistant as well.

3. Keep conversation history

Enables the assistant to remember earlier messages in a session. Useful for context-sensitive conversations (e.g., interactive troubleshooting, revising). For normal chatbots you probably want this. For assistants used in workflows, its normally better to disable.

4. Model

Choose the underlying LLM. Common options:

  • gpt-oss:20b - A fairly fast model. 20 Billion parameter version of the GPT open-source model.
  • gpt-oss:120b - A much more complex version of the 20 billion gpt-oss model.
  • gemma3:7b - A fairly reponsive, but still high quality model.
Tip: Start with the mini model; if the assistant feels "dumb" or inconsistent, upgrade to a larger model.

5. Temperature

Controls randomness of the output.

  • 0.0 - deterministic, very safe.
  • 0.2-0.5 - balanced, good for factual answers.
  • 0.6-1.0 - creative, useful for brainstorming or storytelling.
Recommendation: For most assistants, start at 0.3-0.5.

6. Top K (Number of Document chunks to use for RAG)

When the assistant uses Retrieval-Augmented Generation (RAG), this number determines how many document snippets it can pull in for context.

  • 0 - no RAG (only the prompt).
  • 5-10 - typical for small knowledge bases.
  • 20-30 - for large, detailed documents.
Tip: Increase Top K if the assistant frequently says "I don't know" but you have relevant docs available.

7. Prompt

The prompt is the assistant's "system message" - it sets the tone, role, and behavior. A well-crafted prompt can dramatically improve responses. 

You are FinanceBot, a helpful assistant that provides quick and accurate financial advice.
- Speak in a friendly tone.
- Keep answers concise (≤ 150 words).
- If unsure, say you're not certain and suggest checking official sources.

Best practices:

  • Specify the role (e.g., "You are a travel agent").
  • Set style guidelines (formal, casual, bullet points).
  • Include constraints (max length, no personal data).

8. Documents

Upload or select documents that the assistant can reference. These are broken into "chunks" and fed to the model during a conversation.

  • Available Documents - all files you've uploaded.
  • Added Documents - those selected for this assistant.
How to use:
  1. Click a document in the left list to add it.
  2. Click the "Title:" button in the right list to remove it.
Tip: Keep the most relevant docs in the assistant's list to reduce token usage and improve accuracy.

9. Quick Checklist

  • ✓ Name - descriptive and short.
  • ✓ Shared - if you want to share this bot with other users.
  • ✓ Keep conversation history - enable for context. For normal chatbots you probably want this. For assistants used in workflows, its normally better to disable.
  • ✓ Model - Experiment! Some models may work better than others for some jobs. Smaller models return results faster.
  • ✓ Temperature - 0.3-0.5 for factual work.
  • ✓ Top K - 5-10 for small docs, more for large.
  • ✓ Prompt - clear role, tone, constraints.
  • ✓ Documents - add relevant files.
