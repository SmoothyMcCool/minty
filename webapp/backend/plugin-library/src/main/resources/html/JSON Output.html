JSON Output

Turn the raw results of your workflow into a single, well‑structured JSON document. The rendered JSON is returned as the final output of the workflow.

What It Does

  • Accepts the ExecutionResult from your workflow.
  • Converts that data into a JSON representation.
  • Produces a single text/json file containing the fully rendered document.

Configuration

The JSON Output task does not require any configuration settings.

Parameter Type Description
No configuration needed for this task.

Accessing Data in Your JSON Output

The JSON document will contain these top‑level keys:

  • startTime – Time the workflow started.
  • endTime – Time the workflow completed.
  • results – Map of step names to lists of result objects.
  • errors – Map of step names to lists of error messages.
{
  "startTime": "2025-12-02T10:15:30Z",
  "endTime": "2025-12-02T10:17:45Z",
  "results": {
    "Data Import": [
      { "id": 1, "value": "foo" },
      { "id": 2, "value": "bar" }
    ],
    "Analysis": [
      { "metric": "count", "value": 42 }
    ]
  },
  "errors": {
    "Data Import": [],
    "Analysis": ["Missing field 'value' in record 3"]
  }
}

Output Format

The task always produces output in text/json.

Summary

Use the JSON Output task whenever you need to export the results of a workflow into a single JSON file. No configuration is required – simply add the task to the workflow and the rendered JSON will be stored as the workflow’s output.
